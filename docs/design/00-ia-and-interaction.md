# 白帝控制台 · 信息架构与交互设计规范 v0

> 本规范由 PRD 全篇逆向提炼（11 个模块并行精读）。核心立场：**白帝不是烛龙的换肤，而是从"一次零信任访问决策的生命周期"重新组织的产品**，视觉走 Arco 原生，交互以"向导建对象、抽屉改对象、提交看影响、实时看态势"为主线。

## 0. 与烛龙的根本区别（一句话）

| 维度 | 烛龙 | 白帝 |
|---|---|---|
| 导航模型 | 顶部「六中心」横条 + 左二级菜单 | **左侧可折叠 Sider（Arco Pro 两级菜单）+ 顶栏面包屑** |
| 视觉 | Claude 黏土橙暖色系 | **Arco 原生 ArcoBlue `#165DFF`** |
| 信息架构 | 概览/身份/资源/策略/网关/审计/系统（功能字典） | 态势/访问者/资源/策略/网关/洞察/系统（**访问决策生命周期**；身份+终端合并、安全基线并入策略） |
| 编辑范式 | 表单整页 | **创建走向导、编辑走抽屉** |
| 策略 | 列表 + 表单 | **继承树可视化 + 提交影响预览 + 仿真器** |
| 后端 | 复用 zhulong-control | **自有 baidi-control（Go）** |

## 1. 视觉系统（Arco 原生）

- **主色** ArcoBlue-6 `#165DFF`（rgb 22,93,255）；浅阶 1=`#E8F3FF` 2=`#BEDAFF`。做法：**移除烛龙的 `--primary-*` 暖橙 override，回归 Arco 默认主题**，组件天然 ArcoBlue。
- **中性/文本/边框/填充**：直接用 Arco token —— `--gray-1…10`、`--color-text-1…4`、`--color-border-1…4`、`--color-fill-1…4`、`--color-bg-1…5`。
- **语义色**：success `#00B42A` / warning `#FF7D00` / danger `#F53F3F` / link ArcoBlue。
- **圆角** 2/4/8px；阴影用 Arco elevation；字体走 Arco 默认（PingFang SC / 系统栈）。
- **布局密度（Arco Pro 基线）**：Sider 220px（折叠 48）、顶栏 56px、内容区 padding 16–20、卡片圆角 8、卡片间距 16、内容最大宽自适应。

## 2. 信息架构（围绕访问决策生命周期）

七大工作域（左 Sider 一级，括号内为二级页面）：

1. **态势 Posture** — 安全监控大屏 · 运行总览（实时三防线、在线会话、就近处置入口）
2. **访问者 Subject** — 用户目录 · 组织与角色 · 认证源与策略 · 终端设备与信任 · 终端合规基线
   - 与烛龙不同：**把"身份"与"终端"合并为统一的"访问者"视角**（零信任里主体 = 身份 × 终端，PRD ch7 认证与 ch9 终端高度耦合：授信设备、设备绑定、自适应认证都跨身份+终端）。
3. **资源 Resource** — 应用发布（隧道/WEB 向导）· 应用授权（应用/用户双视角）· 权限审批 · 对象库
4. **策略 Policy** ← 产品灵魂 — 全局策略 · 用户策略（继承树）· 安全基线（应用防护 / 上线准入）· 策略仿真器 · 影响预览
   - 与烛龙不同：**把"安全基线"并入"策略"**（基线本质是"访问决策规则"，与访问策略同源）。
5. **网关 Gateway** — 区域与节点 · SSL 接入 · IPSec 组网 · NAT · 虚拟 IP 池 · SPA 服务隐身（网关内建配置）
6. **洞察 Insight** — 统一日志 · 高级查询与导出 · 审计链 · 日志外送（Syslog / 分析中心）· 流量镜像
7. **系统 System** — 管理员三权分立 · 证书与密钥 · License · 集群 · 升级 · 备份恢复 · 开放平台 · 门户定制

## 3. 通用交互范式（10 个 pattern，全产品统一复用）

> 这些 pattern 从 PRD 各章「X.5 交互要点 + X.6 验收」反复出现的诉求提炼，是白帝"交互细节与烛龙完全不同"的落点。

- **P1 创建走向导**：分步/分支向导建对象 —— 目录接入、应用发布、策略创建、SPA 代际配置、集群组建、IPSec 组网、系统升级。条件分支按类型动态显隐字段。
- **P2 编辑走抽屉**：列表行操作打开**右侧 Drawer** 编辑，不整页跳转；表单状态不丢。
- **P3 继承可视化**：策略页常驻"继承自 组X › 父 › 根"链路 + 「继承 / 自定义」徽章；首次覆写弹**打破继承确认 + 30s 一键撤销**。
- **P4 提交前影响预览**：保存前弹"受影响 N 个用户 / 平台分布 / 与全局策略冲突矩阵 / 风险分级"，勾选确认方可提交。
- **P5 实时池化卡片**：左列表 + 右详情，状态灯 + 心跳时戳，轮询/SSE 局部刷新而非整页。
- **P6 可视化规则/条件构建器**：AND/OR/NOT 行式编辑 + 拖拽排序 + 右侧实时求值预览（替代手写 JSON / 长 checkbox 墙）。
- **P7 风险分级配色**：红=锁定/强制下线、橙=重大客户端变更、黄=轻微；高危操作（关 HTTP/SSH、恢复出厂、集群切主、强制升级）走风险复核弹窗 + 影响说明 + 时间预估。
- **P8 拓扑可视化**：网络接口/集群/资源-网关映射用 SVG 拓扑，节点与链路状态色编码。
- **P9 时间线/状态机**：终端信任生命周期、EDR 风险→权限收缩 timeline、灰度处置倒计时（理念：**最小误杀、最大可恢复**，处置先给"建议方案 + 立即/30min/待审批"选项而非直接断网）。
- **P10 聚合头 + 批量**：列表顶部聚合统计卡（如进程"待研判/已信任/已不信任"计数）+ 批量操作 + 成功/失败明细计数。

## 4. 自有后端

- **baidi-control**（Go，stdlib，`:8090`）—— 控制中心管理 API，独立于烛龙 `zhulong-control`。
- console 经 vite `/api` 反代到 `:8090`；未起后端时页面降级内置 mock。
- 首版 endpoints：`GET /healthz`、`GET /api/v1/meta`、`GET /api/v1/overview`（态势聚合）。模块 API 随页面逐步补齐，`store.Store` 接口先内存、后 SQLite/PG。

## 5. 落地节奏

1. **基座**（本轮）：Arco 原生 design token + Arco Pro Sider 壳 + 新 IA 导航 + 态势页（live 接 baidi-control）+ ComingSoon 占位。
2. **旗舰页**：策略继承编辑器、应用发布向导、访问者目录 —— 树立三个核心 pattern（P3/P1/P5）。
3. **逐模块铺开**：按七大工作域补齐，每页复用 P1–P10。烛龙 fork 的旧页留在 git 历史（commit `a93f0b4`）仅作需求参照，不并入白帝设计。
